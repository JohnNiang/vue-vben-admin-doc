# 引入外部模块

引入第三方组件的前提是需要对Vue3，目前支持的组件库和组件并不是很多。目前支持的 UI 库有Ant-Design-Vue (opens new window)、element-plus (opens new window)。组件支持也不是很多。

在 Vite 项目中,如果依赖遵循 esm 规范,直接引入即可。如果不是,以echarts为例子

  1. 安装 echarts
yarn add echarts
1
  1. 在vite.config.ts内配置
{
  optimizeDeps: {
    include: [
      'echarts',
    ],
  },
}
1
2
3
4
5
6
7

这样才能在 vite 内中使用 echarts 模块。

如果我们需要引入echarts的地图模块，以前的做法是

import echarts from 'echarts';
import 'echarts/map/js/china';
1
2

控制台会抛出如下错误

[vite] Avoid deep import "echarts/map/js/china" (imported by /@/views/demo/echarts/Map.vue)
because "echarts" has been pre-optimized by vite into a single file.
Prefer importing directly from the module entry:
1
2
3

因为我们无法从 echarts 内直接导出 china 变量。所以需要单独引入指定包

// 是没有这个变量的。所以这样写行不通
import { china } from 'echarts';
1
2

此时我们在vite.config.ts内加入echarts/map/js/china重新运行即可。

{
  optimizeDeps: {
    include: [
      'echarts',
      'echarts/map/js/china',
    ],
  },
}
1
2
3
4
5
6
7
8

TIP

optimizeDeps 是用于优化依赖配置。

如果你发现你使用的依赖在 vite 报错，可以往optimizeDeps.include添加相应的包即可
